# MoTo-CARE 代码审查：是否达到发论文地板

## 结论

当前达到的是 **方法原型地板**，还没有达到 **论文投稿实验地板**。

它已经具备可以继续推进的核心机制：

```text
patch/token features
    -> topology-aware adaptive assignment
        -> region tokens
            -> topology prediction loss
                -> optional molecule-region contrastive alignment
```

这说明它不是普通 concat，也不是空壳模型。

## 已发现并修正的问题

1. padding token 原本会参与坐标归一化，可能影响真实 batch 中的空间分配。
2. padding token 原本会影响 assignment entropy 的平均值。
3. 已改成 padding-aware coordinate normalization 和 padding-aware entropy。

## 仍然不够论文地板的地方

1. 拓扑描述还是简化版 prior，不是完整 persistent homology。
2. 还缺真实 WSI feature + coords + cluster topology descriptor 的标准数据脚本。
3. 还缺消融实验：

```text
without topology prior
without topology loss
without molecular alignment
CARE-style assignment only
mean pooling baseline
```

4. 还缺可视化：

```text
region assignment heatmap
topology prediction error per region
pathway-to-region attention
```

## 论文地板判断

可以作为论文项目继续做，但现在还不能说已经能发。最低还需要：

1. 接入至少一个真实 TCGA/公开数据集；
2. 做 image + topology 的初步结果；
3. 证明 topology prior/loss 至少带来稳定提升；
4. 输出 region 可视化图。
