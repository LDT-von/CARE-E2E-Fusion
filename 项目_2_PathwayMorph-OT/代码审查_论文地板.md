# PathwayMorph-OT 代码审查：是否达到发论文地板

## 结论

当前达到的是 **可运行创新模块地板**，还没有达到论文实验地板。

它已经有独立机制：

```text
morphology-topology atoms
    -> pathway tokens
        -> unbalanced Sinkhorn optimal transport
            -> transport-weighted prediction
```

这比普通 cross-attention 更有论文故事。

## 已发现并修正的问题

1. 原来的 OT kernel 可能因为 cost 过大而数值下溢。
2. 已加入 batch-wise cost shift 和 exponent clamp，提高 Sinkhorn 稳定性。
3. 原来的 topology-pathway cost 直接截取 pathway hidden 前几维，语义不合理。
4. 已加入 `topology_to_hidden` 投影，让 topology 和 pathway 在同一 hidden space 比较。

## 仍然不够论文地板的地方

1. 需要真实 pathway activity 构建脚本，不能直接拿随机 pathway token。
2. 需要严格避免标签泄漏，比如不能输入目标 mutation 再预测同一 mutation。
3. 需要 baseline：

```text
WSI-only
omics-only
concat fusion
cross-attention fusion
balanced OT
unbalanced OT
```

4. 需要解释性评估：

```text
top pathways per region
top regions per pathway
transport sparsity
transport heatmap on WSI
```

## 论文地板判断

原创性比 MoTo-CARE 更强，但数据门槛也更高。没有 paired WSI + omics/clinical 数据前，不建议作为第一主线。
